# 提交规范

候选人需在 2 天内提交 Git 仓库地址或压缩包。

## 1. 必交内容

```text
1. 源代码
2. README.md
3. 运行说明
4. 功能说明
5. 测试说明
6. AI 使用说明
7. 过程记录
8. 自我评价
```

## 2. README.md 建议结构

```markdown
# 项目说明

## 1. 题目方向
前端方向 / 后端方向

## 2. 功能清单

## 3. 技术栈

## 4. 如何运行

## 5. 如何测试

## 6. 核心设计说明

## 7. 已完成内容

## 8. 未完成内容及原因

## 9. AI 使用说明

## 10. 后续优化方向
```

## 3. 过程记录要求

建议提交 `process.md`，记录：

1. 需求理解。
2. 任务拆解。
3. 技术方案选择。
4. API / 数据 / 页面 / 状态设计思路。
5. AI 使用过程。
6. 遇到的问题。
7. 调整记录。
8. 验证记录。

## 4. AI 使用记录要求

建议提交 `ai-usage.md`，记录：

| 字段 | 说明 |
|---|---|
| 使用工具 | ChatGPT / OpenCode / Cursor / Claude Code 等 |
| 使用场景 | 需求分析 / 代码生成 / 测试 / 调试 / 文档 |
| Prompt 或 Skill | 可以摘要描述，不要求暴露隐私 |
| AI 生成内容 | 哪些代码、文档、测试由 AI 辅助完成 |
| 人工修正 | 修改了哪些 AI 生成问题 |
| 效果评价 | 哪些有效、哪些无效、遇到什么问题 |

## 5. API 设计说明要求

无论前端还是后端方向，都建议提交 `api-design-proposal.md`。

后端方向：说明实际设计的 API、请求响应、错误处理和状态码。

前端方向：说明你期望后端提供哪些 API，以及当前 Mock 与真实后端如何替换。

## 6. 测试说明要求

建议提交 `test-plan.md`，说明：

1. 测试了哪些核心业务规则。
2. 如何验证状态流转。
3. 如何验证澄清问题阻断规则。
4. 如何验证 AI 辅助能力。
5. 有哪些未覆盖风险。

## 7. 重要说明

本题允许使用 AI，但不接受完全无法解释的 AI 生成结果。面试复盘时，候选人需要能够说明关键代码、设计取舍和验证方式。
